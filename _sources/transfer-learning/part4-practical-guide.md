# 第四部分：实操指南

本章聚焦实际应用中的关键决策点和常见陷阱。

## 4.1 数据集相似性判断

### 4.1.1 图像领域

| 目标领域 | 迁移难度 | 建议 |
|----------|----------|------|
| 自然图像分类 | 低 | 特征提取或轻微微调 |
| 医学影像 | 中等 | 医学专用预训练模型 + 强数据增强 |
| 卫星遥感 | 中等 | 遥感专用预训练模型 |
| 工业缺陷 | 高 | 特征提取 + 数据增强 + 强正则化 |

### 4.1.2 任务粒度

- **粗粒度分类**（猫 vs 狗）：域相似度高，特征提取即可
- **中等粒度**（犬种分类）：部分微调
- **细粒度分类**（鸟类物种）：全量微调 + 数据增强

### 4.1.3 实用判断方法

1. **直接测试**：用预训练模型提取特征，训练简单分类器，性能>70%说明相似度高
2. **对比实验**：同时尝试特征提取和微调，选择验证集性能更好的策略

## 4.2 常见陷阱与对策

### 4.2.1 灾难性遗忘

模型在目标任务上微调后，遗忘预训练阶段学到的通用知识。

**缓解策略**：
1. 使用较小学习率（预训练权重：1e-5 到 1e-4）
2. 分层学习率（浅层更小lr）
3. 使用PEFT方法（LoRA、Adapter）

### 4.2.2 过拟合

小数据集上微调大模型，模型"记住"训练样本而非学习规律。

**识别信号**：训练损失下降但验证损失不下降或上升

**应对策略**：
1. **数据增强**：随机裁剪、翻转、旋转、颜色抖动
2. **早停**：监控验证集性能，不再提升时停止
3. **正则化**：权重衰减（weight_decay=1e-4）、Dropout
4. **减少可训练参数**：特征提取或PEFT

### 4.2.3 学习率设置不当

- **学习率过大**：损失震荡不收敛，破坏预训练权重 → 降低学习率
- **学习率过小**：收敛极慢，陷入局部最优 → 增大学习率，使用调度器

```python
# 推荐配置
scheduler = torch.optim.lr_scheduler.ReduceLROnPlateau(
    optimizer, mode='min', factor=0.1, patience=5
)
```

### 4.2.4 预处理不一致

预训练模型对输入数据有特定预处理要求，必须匹配：

```python
# ImageNet标准化参数
transform = transforms.Compose([
    transforms.Resize(256),
    transforms.CenterCrop(224),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406], 
                         std=[0.229, 0.224, 0.225])
])
```

**关键检查点**：
- [ ] 输入尺寸匹配（通常224×224）
- [ ] 使用正确的归一化参数
- [ ] 颜色通道顺序正确（RGB）

## 4.4 高级技巧

### 测试时增强（TTA）

对测试图像进行多轮增强，取平均预测，可提升1-3%性能。

### 集成学习

使用多个预训练模型集成：
- 不同架构（ResNet + EfficientNet + ViT）
- 不同预训练数据

### 持续学习

模型需持续适应新任务时：
- 使用PEFT方法（LoRA、Adapter）
- 考虑经验回放或EWC等持续学习方法 {cite}`kirkpatrick2017overcoming`

## 4.5 本章小结

**数据集相似性判断**：内容相似性、任务粒度、量化方法

**常见陷阱**：
- 灾难性遗忘 → 小学习率、PEFT
- 过拟合 → 数据增强、早停、正则化
- 学习率问题 → 分层学习率、调度器
- 预处理 → 使用预训练时的预处理参数

**核心思想**：站在巨人的肩膀上，善用预训练知识。

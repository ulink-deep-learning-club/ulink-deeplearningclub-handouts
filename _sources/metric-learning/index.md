# 深度学习中的度量学习

**作者**：Anson，深度学习社  
**最后更新**：2025-03-07

## 摘要

度量学习（Metric Learning）是深度学习中的重要范式，其核心思想是通过学习合适的嵌入空间和距离度量，使相似样本靠近、不相似样本远离。这种方法突破了传统分类方法的封闭集限制，支持开放集识别、小样本学习和增量学习，在人脸识别、图像检索、行人重识别等领域取得了巨大成功。

本讲义系统介绍度量学习的理论基础与实践技术：阐述度量学习相对于传统分类方法的优势与核心动机；从监督方式、变换性质、局部性等多个维度梳理度量学习的分类体系；深入讲解经典损失函数（对比损失、三元组损失）与现代损失函数（N-pair、ArcFace、Proxy Loss）的设计原理；最后提供完整的PyTorch实现、采样策略、评估方案和常见问题解决方案。

## 为什么学习度量学习？

### 技术价值

```{admonition} 度量学习的重要性
:class: tip

1. **开放集识别能力**：处理训练未见过的类别，实际应用中的常见需求
2. **小样本学习潜力**：通过度量迁移，少量样本即可实现高效识别
3. **相似度推理基础**：为推荐系统、检索系统提供底层技术支持
4. **研究热点持续**：顶级会议（CVPR、ICCV、NeurIPS）每年有大量相关论文
```

### 应用场景

度量学习已广泛应用于以下领域：

- **计算机视觉**：人脸识别、行人重识别、图像检索、细粒度分类
- **推荐系统**：商品相似度推荐、用户兴趣匹配
- **自然语言处理**：文本相似度、语义检索
- **医学影像**：病灶检索、病例相似性分析
- **生物识别**：指纹、虹膜、步态识别

### 与其他主题的关系

度量学习并非孤立的知识点，而是与其他深度学习技术紧密关联：

- **与迁移学习**：度量学习常利用预训练模型迁移通用视觉知识
- **与对比学习**：两者共享"拉近正样本、推远负样本"的思想，可相互借鉴
- **与注意力机制**：注意力可用于增强度量的判别能力
- **与自监督学习**：对比自监督学习为度量学习提供好的特征初始化

## 常见问题

```{admonition} Q1：度量学习和分类任务有什么区别？
:class: note

| 维度 | 分类（Classification） | 度量学习（Metric Learning） |
|------|------------------------|----------------------------|
| **学习目标** | 预测样本属于哪个预定义类别 | 学习样本间的相对关系 |
| **输出** | 类别概率分布 | 特征向量 + 距离 |
| **样本组织** | 单样本 + 类别标签 | 样本对/三元组 + 相似性标签 |
| **新类别处理** | ❌ 需要重新训练 | ✅ 直接计算距离 |
| **开放集识别** | ❌ 不支持 | ✅ 支持 |

**核心区别**：分类要求样本靠近类中心；度量学习要求同类样本聚集，但不必重合。
```

```{admonition} Q2：应该使用什么损失函数？
:class: tip

**初学者路线**：
1. **Triplet Loss**（首选）
   - 概念直观，易于理解
   - 大多数场景下表现良好
   - 配合Semi-Hard Mining使用

**进阶选择**：

| 场景 | 推荐损失函数 | 原因 |
|------|-------------|------|
| 数据量小 | Contrastive Loss | 简单高效 |
| 分类多、每类样本多 | N-pair Loss | 充分利用batch内样本 |
| 大规模多类任务 | ArcFace | 角度间隔，性能优异 |
| 类别数极大(>10K) | Proxy Loss | 与代理比较，计算高效 |
```

```{admonition} Q3：batch size多大合适？
:class: note

**基本原则**：
- 每类至少**4个样本**
- Batch size建议**32-128**

**为什么需要大batch？**

| Batch Size | 负样本数量 | 效果 |
|-----------|-----------|------|
| 16 | 12 | 负样本不足，训练不稳定 |
| 32 | 28 | 基本可用 |
| 64 | 60 | 效果良好 |
| 128+ | 124+ | 效果最佳，但显存压力大 |

**实践建议**：
- 显存允许的情况下，越大越好
- 使用P-K采样（P类，每类K个样本）
- 例如：P=16, K=4, Batch=64
```

```{admonition} Q4：如何判断训练是否成功？
:class: important

**关键指标**：

| 指标 | 健康状态 | 异常状态 | 解决方案 |
|------|---------|---------|----------|
| **Training Loss** | 逐渐下降 | 不降或震荡 | 检查学习率、采样策略 |
| **Validation Recall@K** | 持续提升 | 停滞或下降 | 可能过拟合，增加正则化 |
| **hardest_positive** | 逐渐减小 | 增大 | 检查margin设置 |
| **hardest_negative** | 逐渐增大 | 减小或不变 | 检查采样策略 |

**t-SNE可视化诊断**：
- ✅ **健康**：同类样本聚集，不同类分离
- ❌ **度量坍缩**：所有点混在一起
- ❌ **欠拟合**：无明显聚类结构
```

## 数学符号说明

本讲义使用以下数学符号约定：

- $x$ 或 $\mathbf{x}$：输入样本
- $y$：类别标签
- $f_\theta(\cdot)$：嵌入网络，$\theta$ 为参数
- $z = f_\theta(x)$：嵌入向量（特征）
- $d(\cdot, \cdot)$：距离函数
- $\mathcal{L}$：损失函数
- $m$ 或 $\alpha$：间隔（margin）
- $B$：批量大小
- $N$：样本总数
- $C$：类别数
- $D$：嵌入维度
